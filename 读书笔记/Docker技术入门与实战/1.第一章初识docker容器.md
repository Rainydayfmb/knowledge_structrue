# 初识docker与容器
## 什么是docker
### docker开源项目背景
docker是基于GO的开源容器项目。docker的构想是实现build,ship and run any app,anywhere,即通过对应用的封装，分发，部署，运行生命周期进行管理，达到应用组装级别的一次封装，到处运行。
## 为什么要使用docker
### docker容器虚拟化的好处
docker通过容器打包应用，解耦应用和运行平台。这意味这在迁移的时候，只需要在新的服务器上启动需要的容器就可以了，无论新旧服务容器，是否同一个平台类型。
### docker在运维中的优势
- 更快速的交付与部署 
- 更高效的资源利用 运行docker 不需要额外的虚拟化管理程序的支持，docker是内核级别的虚拟化，可以实现更高的性能，同时对资源的额外需求很低。与传统虚拟即方式相比，docker的性能要提高1～2个数量级。
- 更轻松的迁移和扩展
- 更简单的更新管理
## docker与虚拟机比较
- docker容器很快
- docker容器对系统资源需求很少
- docker通过类似git设计的理念的操作来方便用户获取、分发和更新应用景象、存储复用，增量更新
- docker通过dockerfile支持灵活的自动化创建和部署机制，以提高工作的效率，并标准化流程。
## docker与虚拟化
虚拟化的技术核心是对资源的抽象，目的是为了在同一个主机上同时运行多个系统或应用，从而提高系统资源的利用率，并且带来降低成本、方便管理和容错容灾等好处。
docker以及其他容器都属于操作系统虚拟化这个范畴，操作系统虚拟化最大的好处就是不需要额外的supervisor支持。docker虚拟化方式之所以有众多的优势，跟操作系虚拟化技术自身的设计和实现是分不开的。
